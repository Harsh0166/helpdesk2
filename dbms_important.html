Important Questions

@Debuggers

1. Definition of DBMS: A Database Management System (DBMS) is software that facilitates the creation, organization, retrieval, and management of data in a structured format...

2. Advantages of DBMS: Data Integrity: Ensures accuracy and consistency of data. Data Security: Controls access to the data to prevent unauthorized usage...

3. Components of DBMS: Data Definition Language (DDL), Data Manipulation Language (DML), Data Query Language (DQL), Transaction Control Language (TCL)...

4. Entities and Attributes: An entity is a real-world object or concept represented in the database. Attributes are properties or characteristics of entities...

5. Tables and Rows: Tables are the fundamental structure for organizing data in a relational database. Rows represent individual records in a table, and columns represent attributes...

6. Primary Key: A primary key uniquely identifies each record in a table. It must be unique for each record and cannot contain NULL values...

7. Foreign Key: A foreign key establishes a relationship between two tables. It refers to the primary key of another table and enforces referential integrity...

8. Normalization: The process of organizing data to minimize redundancy and dependency. Normal forms (1NF, 2NF, 3NF, BCNF) help achieve a well-organized database...

9. ACID Properties: Atomicity, Consistency, Isolation, and Durability ensure the reliability of transactions. For example, in a banking system, transferring funds from one account to another must maintain these ACID properties...

10. Indexes: Indexes improve the speed of data retrieval operations. Clustered indexes determine the physical order of data, while non-clustered indexes do not...

11. Joins: SQL JOIN operations (INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL JOIN) combine rows from multiple tables based on related columns...

12. Views: A view is a virtual table derived from the result of a SELECT query. It provides a way to present a specific perspective of the data to users...

13. Stored Procedures and Triggers: Stored Procedures are precompiled sets of SQL statements executed with a single call. Triggers are sets of instructions that automatically execute in response to specific events...

14. Database Design Considerations: Consider factors such as normalization, denormalization, data types, and performance trade-offs when designing a database schema...

15. Database Optimization: Involves query optimization, index tuning, and caching to enhance database performance. Techniques such as proper indexing and query design can significantly impact performance.
